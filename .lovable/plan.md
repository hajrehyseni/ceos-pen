
## Goal

Give you three high-quality draft choices every day on the Today view, and restructure the desktop layout so the right column shows **Spend first, AI News second** underneath it.

## Changes

### 1. Three drafts on the homepage (not just one hero)

Today the `HeroDraftCard` shows a single top-ranked draft. Replace it with a **"Today's picks" stack** that surfaces the top **3 drafts** ranked by the existing `scoreRank` (virality + engagement + verification).

- **Slot 1 (Hero)** — top-scored draft, full `HeroDraftCard` treatment (first comment, CTA, Visual Studio, actions).
- **Slots 2 & 3 (Alternates)** — compact cards showing: pillar tag, hook (first 2 lines), score chips (virality, engagement, verification, scorecard-attached ✓). Tap to expand into the hero slot (swap in place) or open the full DraftCard sheet.
- If fewer than 3 drafts exist, show what's available + a "Generate another" button that calls `generate-draft` so you always have choice.
- Each alternate card shows a small "Make this the pick" button that promotes it into the hero position for the day (local UI state — no schema change).

### 2. Layout restructure

**Desktop (≥1024px):** two-column editorial desk stays, but the right column reorders:

```text
┌─────────────────────────┬──────────────────┐
│ Greeting                │  CostStrip       │
│ Today's picks (1 hero   │  (Today/Week/    │
│  + 2 alternates)        │   Month spend)   │
│                         │                  │
│                         │  AI News (top 3) │
│                         │                  │
│                         │  Agent status    │
└─────────────────────────┴──────────────────┘
```

**Mobile (<1024px):** stacked order becomes:
1. Greeting
2. Today's picks (hero + 2 alternates)
3. CostStrip
4. AI News (compact, top 3)
5. Agent status footer

(Currently on mobile News sits above Spend — this flips them, matching your request.)

### 3. Ensure 3 quality drafts are actually available

Right now drafts are only generated by the 7:30 AM UTC cron. Add a lightweight "Top up drafts" affordance on the Today view: if `queuedDrafts.length < 3`, show a subtle inline button that invokes `generate-draft` to backfill. No auto-loop — user-triggered only.

## Files touched (frontend only)

- `src/pages/Index.tsx` — reorder desktop `aside` (CostStrip → CompactNewsList → AgentStatusFooter) and mobile stack (picks → CostStrip → News → Agent). Pass top-3 drafts into new picks component.
- `src/components/mobile/HeroDraftCard.tsx` — accept a "featured" draft prop instead of picking `drafts[0]` internally, so the parent controls which draft is hero.
- **New:** `src/components/mobile/TodaysPicks.tsx` — renders hero + 2 alternates, handles swap-in-place, "Top up" button when <3 drafts.
- **New:** `src/components/mobile/AlternateDraftCard.tsx` — compact card for slots 2 & 3 (pillar, hook preview, score chips, promote button).

No backend, schema, or edge-function changes. No changes to draft generation logic or scoring.

## Out of scope

- Changing the scoring algorithm itself (still uses existing `virality_score` + `engagement_estimate` + `verification_status`).
- Auto-generating drafts on page load (opt-in button only, to avoid runaway API spend).
- Changes to Drafts / Published / Analytics tabs.
